Blog Forms
05 Jun 2013

Blogging is kind of annoying. I’ve tried using many blogging platforms, including Wordpress, Blogger, Tumblr, and Movable Type, but none of these really ever worked. There’s one goal of blogging: to post substantive posts in consistent periods of time. I’ve never really been able to do this, and part of the reason is that it just isn’t fun to write posts on these platform.

Look at Wordpress, for example. The good thing about Wordpress is that it has every option that someone would ever need in customizing a blog. The bad thing is that it has everything. It’s nice that it’s easy to add some pictures, edit the permalink, or change the published date, but I’m never going to need to add a poll or a contact form or connect the post to my nonexistent LinkedIn or turn off likes for just one post or do some other random specific-use things1. I just want to write.

I also want my blog posts to be at least half the size of an article and with a more personal bent. This is where Tumblr doesn’t work for me. There are many people who use Tumblr as a blogging platform – here’s a few music critics (kind of) doing so – but that’s not what it’s intended for. Tumblr is fantastic for posting/reposting gifs and for reactionary comments, but as a short-form writing platform, it’s terrible.

Fortunately, some people have realized what’s wrong with blogging. Dustin Curtis created his Svbtle platform/magazine a year ago, and more recently Medium popped out of nowhere. These two sites have beautiful post editors. The ten minutes that I interacted with the Medium platform was pretty amazing due to how clean everything was and how everything just worked. My only problems with these two sites are that they’re exclusive and all blogs look the same. I just want that clean easy-blogging interface, but it has to be tightly connected with this rigid (albeit very nicely designed) frontend output.

I’ve been trying to work on making some kind of nice blogging interface during my free time. Here’s the project on GitHub; it’s currently not completely working right now as I’ve been using the project as a Javascript framework learning tool more than anything. The interface is really simple; all there is to each post is a title, the post body, and adding tags. The post body uses Markdown to parse the content (like everything seems to do nowadays). It’s not completely thought out yet, and I might just end up changing everything about it tomorrow, so…

Meanwhile, I’m just using Jekyll for this website. I’m writing my posts in a text editor with Markdown, and honestly, that’s enough. I might just stick with this.

  1. Two more things about Wordpress:

    1. It has a distraction-free writing mode that I just noticed. It’s a bit better, but…nah.
    2. Wordpress is written in PHP, so it’s not very fun if there’s something specific I want to change that’s not in their hundreds of options.
